Construyendo sitios estáticos con Jekyll

Dr. Jekyll & Mr. Ruby

Perpretado por Sergio Sánchez

Un sitio web dinámico puede tener los siguientes inconvenientes:

  • Complejidad
  • Costo y Rendimiento
  • Obsolecencia

Las páginas estáticas, por el contrario, tienen los siguientes problemas:

  • Debes conocer HTML para poder actualizarlas
  • Dificil jerarquizar un proyecto
  • El principio DRY es dificil de cumplir

¿Qué es Jekyll?

  • Es un motor de generación de sites estáticos creado por Tom Preston Werner
  • Es una gema de ruby
  • Crear contenidos con markdown y plantillas con HTML5/CSS3/JS
  • Altamente personalizable, escalando mediante plugins

    • Tom es también co-fundador de GitHub

¿Por qué Jekyll?

  • Es simple, versátil... y funciona!
  • Todo queda bastante ordenado
  • Se puede comprometer en Git
  • Tiene su propio servidor para probar el sitio
  • Olvidarte de buscar un hosting adecuado.
  • Rapidez de respuesta
  • Puedes usar dominio propio
  • Seguridad

¿Y todo esto cómo funciona?

Simplemente, el motor Jekyll procesará todo archivo que contenga un bloque YAML como un archivo especial. Este bloque YAML debe estar al principio del archivo y se identifica mediante 3 guiones, tal que así: 

				 --- 
				 layout: post 
				 title: blogueando like a monster 
				 ---

El ingrediente del monstruo... LIQUID

Liquid es un lenguaje para plantillas, creado por Shopify y escrito en ruby. Permite incorporar lógica en las plantillas pero que dicha ĺógica no se ejecute en el servidor. Jekyll utiliza el motor liquid para renderizar la página en el layout definido.

Generando nuestro sito

Publicar nuestro blog en Git Hub

			git push origin gh-pages 
			
			Visitar: "http(s)://username.github.io"

Asociar un dominio propio

Para asociar un dominio propio, hemos de crear el archivo CNAME en la raiz del sitio y agregar al mismo nuestro dominio:

			squevedo.name
